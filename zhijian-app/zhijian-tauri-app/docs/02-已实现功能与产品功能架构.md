### 小结

- **已实现并可直接迁移的 UI / 交互**：标题栏拖拽、颜色切换、Markdown 编辑、置顶/新建/关闭按钮、键盘快捷键 —— 这部分在 Tauri 中只需把 `-webkit-app-region: drag` 替换为 `data-tauri-drag-region`。
- **需要替换为 Tauri 原语的系统能力**：窗口创建、置顶、单实例、托盘、持久化到磁盘。
- **需要在多窗口架构下重新设计**：搜索、聚焦、自动整理 —— 这些 Web 版是"单页多卡片"的实现，迁移到"多独立窗口"需切换思路。

## 2. 产品功能架构

### 2.1 分层视图

```
  ┌──────────────────────────────────────────────────┐
  │  表现层（Presentation）                            │
  │   ├── HTML + CSS（色彩 / 字体 / 纹理 / 布局）      │
  │   ├── 内容编辑器（contenteditable + Markdown）     │
  │   └── 悬浮操作区（颜色切换 / 置顶 / 新建 / 关闭）   │
  ├──────────────────────────────────────────────────┤
  │  交互层（Interaction）                             │
  │   ├── 拖拽标题栏移动窗口                          │
  │   ├── 系统原生窗口缩放                             │
  │   ├── 键盘快捷键（Ctrl+N / Ctrl+W）                │
  │   └── 右键托盘菜单                                │
  ├──────────────────────────────────────────────────┤
  │  应用服务层（App Service）                          │
  │   ├── 便利贴生命周期管理（创建/关闭/恢复）           │
  │   ├── 单实例锁（防止多进程重复启动）                  │
  │   ├── 持久化服务（读取/写入 note.json）             │
  │   └── 系统集成（托盘 / 通知 / 自启 / 关联协议）      │
  ├──────────────────────────────────────────────────┤
  │  运行时（Runtime）                                 │
  │   └── Tauri v2（Rust backend + Windows WebView2）  │
  └──────────────────────────────────────────────────┘
```

### 2.2 数据模型

每张便利贴被抽象为一个 `Note` 对象，字段设计：

```ts
interface Note {
  id: string;            // 雪花或 UUID（创建即生成）
  color: ColorToken;     // 'cream' | 'blue' | 'pink' | 'green' | 'lavender' | 'yellow'
  content: string;       // 渲染后的 HTML
  plainText: string;     // 纯文本，用于搜索过滤
  pinned: boolean;       // 置顶
  collapsed: boolean;    // 收起（保留字段，v0.1 可未实现）
  clickThrough: boolean; // 鼠标穿透
  rect: {                // 位置与尺寸
    x: number;
    y: number;
    width: number;
    height: number;
  };
  reminder?: string;     // ISO 8601 datetime
  createdAt: number;     // timestamp ms
  updatedAt: number;     // timestamp ms
}
```

存储文件：`{appDataDir}/notes.json`（单文件，便于备份 / 迁移）

## 3. IPC 命令契约

### 3.1 窗口操作（由每张便利贴窗口的前端 → 后端）

```ts
// 前端调用方式：await invoke('command_name', payload)

create_note(payload?: { color?: string, content?: string, x?: number, y?: number }): Promise<WindowLabel>
close_note(): Promise<boolean>
pin_note(pinned: boolean): Promise<boolean>
minimize_note(): Promise<boolean>
set_click_through(flag: boolean): Promise<boolean>
get_window_rect(): Promise<{x, y, width, height}>
set_window_rect(rect: {x, y, width, height}): Promise<boolean>
```

### 3.2 数据持久化（每张便利贴窗口内部调用）

```ts
save_note(note: Note): Promise<boolean>
delete_note(id: string): Promise<boolean>
load_all_notes(): Promise<Note[]>     // 仅在主进程启动时调用一次
```

### 3.3 应用级（托盘 / 通知）

```ts
show_all_notes(): Promise<boolean>
hide_all_notes(): Promise<boolean>
quit_app(): Promise<boolean>
show_notification(title, body): Promise<boolean>
```

### 3.4 单实例

由 `plugin-single-instance` 处理，首次启动若检测到已有实例，仅向其发送"新建一张便签"消息。

## 4. 迁移到 Tauri 的改造点

### 4.1 替换清单

| 原 Electron 实现 | Tauri 实现 |
|---|---|
| `BrowserWindow({frame:false, transparent:true})` | `WindowBuilder::new().decorations(false).transparent(true)` |
| `-webkit-app-region: drag` | `data-tauri-drag-region` 属性 |
| `win.setAlwaysOnTop(true)` | `window.set_always_on_top(true)` |
| `win.setIgnoreMouseEvents(true, {forward:true})` | `window.set_ignore_cursor_events(true)` |
| `app.requestSingleInstanceLock()` | `tauri-plugin-single-instance` |
| `Tray` / `Menu` | `tray-icon` crate（Tauri v2 内建） |
| `localStorage` 保存内容 | Rust `std::fs` 写入 `appDataDir/notes.json` |
| Electron 快捷键 `globalShortcut` | 前端 `keydown` 即可覆盖基础场景 |
| 打包 electron-builder | `tauri build` → NSIS / MSI / Portable |

### 4.2 多窗口模式在 Tauri 中的推荐实现

```
主进程 main.rs 启动流程：

  1) 初始化 tauri::Builder
  2) 读取 notes.json（若不存在则创建空白模板）
  3) 对每张 Note 调用 WindowBuilder 新建一个 Webview 窗口
        窗口标签 = "note-{id}"
        每个窗口都 load 同一个 index.html
        前端通过 window.__TAURI__.window.getCurrent().label 识别自己是哪张
  4) 若无任何便签，创建一张欢迎便签
  5) 注册系统托盘
  6) 启动完成
```

### 4.3 前端改造要点

- 移除所有 `window.zhijian.*` 调用，替换为 `invoke('create_note', {...})`
- 把 Electron 的"通过 URL query 传入初始 color"改为通过 `window.label` + invoke 获取自己的 `Note` 数据
- `-webkit-app-region: drag` → `<div data-tauri-drag-region>...</div>`
- 其他视觉 CSS 100% 可复用

## 5. 产品架构原则（随项目扩展）

1. **本地优先（Local-first）**：数据存本地、可离线、可备份；云端同步永远是附加项，不破坏本地可用性。
2. **便签即窗口（One-Note-One-Window）**：每个便利贴对应一个桌面窗口，是核心交互原子，不做统一"主窗口"。
3. **静默保存**：编辑即自动保存，不设置"保存"按钮。
4. **最小权限**：Tauri allowlist 仅开放 `window | fs | notification | globalShortcut` 实际用到的范围。
5. **保持轻量**：拒绝非必要依赖；打包后 <15MB 作为硬性指标。
