# 纸间 v0.2 · 小版本需求文档

> 版本：v0.2 "条目便利贴：条目拖拽排序
> 负责人：纸间团队
> 状态：需求已冻结，开始实现

---

## 1. 需求背景

当前每张便利贴是一个自由文本编辑区。
用户反馈：在记录待办事项、购物清单、任务分解时，希望能按"条目"为单位调整顺序，
而不是手动复制粘贴去挪位置。

## 2. 目标与范围

本次小版本仅在**单张便利贴**内增加"条目"级别的拖拽排序**能力，
不涉及多便利贴之间的数据模型变更（窗口本身仍可自由拖动/缩放/置顶，不改动）。

## 3. 核心用户故事

**US-01** — 切换到条目模式**
> 作为用户，我希望点击便签顶部的"☰ 列表"切换按钮，
使当前便利贴从纯文本模式切换为**条目模式**，
以便我按条录入多条可排序的任务。

**US-02** — 添加 / 修改条目**
> 作为用户，我希望在条目模式下看到一个回车就新添加一条，
并且可以像在每条的正文直接编辑，
以便我快速记录多条内容。

**US-03** — 拖动排序**
> 作为用户，我希望按住每条左侧的"把手（☰），把某条拖到第一、第二、
或其它任何位置，
以便我随时调整顺序。

**US-04** — 持久化**
> 作为用户，我希望条目顺序、每条的完成状态（可选）被自动保存到本地，
以便下次启动窗口时内容不变。

**US-05** — 切换回文本模式时保留现有内容**
> 作为用户，我希望切换回文本模式时，
所有条目应以换行合并为纯文本，
每条一行，
以便我在两种模式之间无损切换。

## 4. 交互规格

### 4.1 模式切换

在标题栏中增加一个按钮（在**按钮：

**视觉：
- 文本模式（默认）：按钮显示 "☰"，hover 提示"切换到条目模式"。
- 条目模式：按钮高亮/填充。

### 4.2 条目模式下的条目外观

每条条目由三部分组成，从左到右：

```
┌────────────────────────────────┐
│ ☰  ☐  这是一条可编辑的文本   │ ← 条目行（整个条目行都是拖到其它位置
├────────────────────────────────┤
│ ☰  ☐  第二条，可拖动到第一  │
│     第一条上方会出现"插入线"   │
└────────────────────────────────┘
```

- **☰ 拖拽把手：仅在 hover 条目或 hover 条目时显示。
- **☐ 完成勾选框**（可选小版本可复选框 v0.2 可选实现）：点击切换完成状态；
  勾选后正文置灰 + 中划线。
- **正文**：点击即可编辑；Enter 新建下一条；Backspace 在空条目的条目中删除本条并将光标条目的空条目可整条。

### 4.3 拖拽交互

- 拖拽过程中：
1. 鼠标按下 **按住条目变半透明。
2. 其它条目位置：在插入位置会显示一条 **2px 高亮插入指示线。
3. 松开鼠标：当前条目落在插入指示线位置。

### 4.4 持久化

- 文本模式：`content` 存 HTML（保留）。
- 条目模式：`items` 存 `[{id, text, done}`。
- `mode` 字段：`"text" | "items"`。

切换策略：
- **→ items** 时，把 `content` 按 `<br>` / `\n` 拆分为条目。
- **→ text** 时，把 items 用 `\n` 合并回文本。

### 4.5 键盘

| 操作 | 按键 |
|------|------|
| 新条目 | Enter（在条目末尾） |
| 删除本条 | Backspace（空条目时） |
| 上移本条 | Alt + ↑ |
| 下移本条 | Alt + ↓ |
| 勾选完成 | Ctrl + Enter |

## 5. 数据结构变更

单张便利贴 `Note` 扩展：

```json
{
  "id": "n1abc",
  "color": "cream",
  "mode": "items",
  "content": "",
  "items": [
    { "id": "i1", "text": "写周报", "done": false },
    { "id": "i2", "text": "回复邮件", "done": false }
  ],
  "rect": { "x": 120, "y": 120, "width": 280, "height": 280 },
  "pinned": false,
  "clickThrough": false,
  "createdAt": 1712345678000,
  "updatedAt": 1712345678000
}
```

为保持对老数据（`mode` 字段，`content` 与 `items` 字段互斥：

- `mode == "text" 时，以 `content` 有效。
- `mode == "items"` 时，以 `items` 为准。

向后兼容：老便利贴没有 `mode` 字段一律按 "text" 默认。

## 6. 非功能需求

- **响应：单张便利贴条目的响应延迟 ≤ 100 条时无卡顿。
- 拖拽动画使用原生 HTML5 Drag & Drop 即可，无需第三方库。
- 与现有 v0.1 的其它窗口/窗口交互行为一致（窗口仍可拖动。
- 保存频率：失焦 + 每 10s 自动保存一次。

## 7. 技术实现要点

### 前端（`src/index.html）

1. 标题栏增加 "☰" 切换按钮（在 pin 与 + 与✕ 之间）。
2. `note-body` 容器：
   - `data-mode="text" 时，行为同 v0.1。
   - `data-mode="items"` 时，渲染 `<ul>` + `<li draggable="true"`。
3. 每条 `<li>` 内含：拖拽把手、复选框、可编辑 `<span`。
4. 使用 HTML5 dragstart / dragover / drop 事件，dragover 时计算插入位置、画指示线。
5. dragend 后重新排序 items 数组并触发 save。

### 后端（`src-tauri/src/note.rs` + `main.rs`）

1. `Note` 结构体增加：
```rust
pub struct Item { pub id: String, pub text: String, pub done: bool }
pub enum Mode { Text, Items }
pub struct Note {
    // ...
    pub mode: Mode,
    pub items: Vec<Item>,
    // ...
}
```
2. `cmd_save_note` 接收 `mode`, `items_json` 参数，`_text`, `done`_text` 字段。

3. ✅ 关闭重启应用重开应用后，条目的顺序与文本一致。
